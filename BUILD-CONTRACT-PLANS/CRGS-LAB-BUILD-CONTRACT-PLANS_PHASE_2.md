### IV. The Deterministic LLM Binding (`core/llm_binding.py`)

To prevent the Generator and Judge from drifting into hallucination, the model connection must be strictly bound. We enforce `temperature=0.0` and utilize LangChain's structured output parsers to guarantee the AI’s response strictly adheres to a JSON schema. If the model outputs conversational fluff, the parser throws an exception and halts execution.

```python
from langchain_core.pydantic_v1 import BaseModel, Field
from langchain_openai import ChatOpenAI

# 1. Define the immutable output schema for the Generator Agent
class HypothesisPayload(BaseModel):
    hypothesis: str = Field(description="The scientific hypothesis text")
    claims: list[str] = Field(description="List of factual claims supporting the hypothesis")
    citations: list[str] = Field(description="Exact reference ID mapped from Neo4j")
    validation_code: str = Field(description="Python code using SymPy to prove mathematical assertions")

# 2. Hard-code the LLM Binding with Zero Temperature
def get_deterministic_generator():
    """
    Binds the LLM to absolute 0 temperature to eliminate stochastic variability,
    enforcing a 100% adherence to the defined Pydantic schema.
    """
    llm = ChatOpenAI(
        model="gpt-4o", # Or local Llama-3 equivalent via vLLM
        temperature=0.0, 
        max_retries=3,
        model_kwargs={"seed": 42} # Force seed for exact reproducibility
    )
    
    # Enforce JSON payload structure natively
    structured_llm = llm.with_structured_output(HypothesisPayload)
    return structured_llm

```
