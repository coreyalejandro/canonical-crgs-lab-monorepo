Build this in Divi. The look ships with it.

You build the structure in Divi. One stylesheet and a few code blocks carry the entire design and the interactions. This page is everything you need on one sheet.

What this is

A redesign of the Qi149 demo into the CRGS Lab identity, in the client's own portfolio design language. The kit gives you a high-fidelity mockup to match, an original logo, one global stylesheet, paste-in code blocks, and a deterministic step-by-step guide.

Pick one build path

Path A · Fastest

Code-module path

Follow docs/03. Content sections are pasted as Code modules. Reproduces the mockup verbatim.

Path B · Editable

Native Divi path

Follow docs/04 and also paste divi-native-supplement.css. Content sections become native Divi modules.

Either way: the three interactive pieces — the six-layer stack, the kernel status bar, and the reviewer-mode control — stay as Code modules. A page builder cannot reproduce them natively.

Build order

  1. Open mockup/index.html in a browser. That is the target.
  2. Add the Google Fonts link to the page head (Divi → Theme Options → Integration).
  3. Paste global-custom-css.css into Divi → Theme Options → Custom CSS.
  4. Upload the logo (Theme Customizer) and set the favicon (Site Identity).
  5. Build the Home sections in order, giving each its CSS ID from the manifest.
  6. Paste the interactive code blocks into Code modules.
  7. Build the navigation menu, publish, and set the page as the site homepage.

Accessibility — please keep these

The client has specific cognitive and visual needs. These are not optional polish.

  • Do not add flashing, strobing, autoplaying, or looping motion.
  • Keep the reduced-motion behavior that is already in the CSS.
  • Keep clear text labels on every control. Do not rely on icon-only buttons.
  • Keep the strong color contrast as delivered.

Definition of done

  • The published Home page matches mockup/index.html.
  • The six-layer stack opens one layer at a time.
  • Reviewer Mode reveals a status pill on every program card.
  • The kernel status bar hides and returns.
  • The logo, favicon, and social card are installed.
  • Every section carries its CSS ID from the manifest.
